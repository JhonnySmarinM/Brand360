# Technical Documentation: Brand360™

## 1. Project overview

Brand360 is an interactive web application designed for entrepreneurs and creatives. It uses a three-step workflow to turn abstract concepts into tangible brand identities by providing:

- **Naming**: Suggestions for brand names based on the sector
- **Slogans**: Impactful phrases that communicate the value proposition
- **Visual concept**: Guidelines for style, typography, and design feel

## 2. System architecture

The application is built as a modern React app using Vite as the build tool, with a \"state machine\"-style model to manage navigation between process phases.

### Prerequisites

- **React.js 18+**: Base framework
- **Vite 5+**: Build tool and dev server
- **Tailwind CSS 3+**: For visual design and animations
- **Lucide React**: Icon library
- **Google Fonts**: Inter and Noto Sans (recommended)

## 3. Flow breakdown (User flow)

### Step 1: Idea configuration (Idea step)

The user enters the business description and value tags.

- **Validation**: A minimum of 5 characters in the description is required to enable generation
- **Keywords**: Dynamic input system that adds tags when pressing Enter
- **Limit**: Maximum of 5 keywords per project

### Step 2: Identity generation (Identity step)

Multiple brand options generated by AI are presented.

- **Plan segmentation**: Results change dynamically according to the selected level (Free, Basic, Premium, Pro)
- **Selection UI**: Cards with active/inactive states to choose the favorite identity
- **Regeneration**: Changing the plan automatically regenerates identities

### Step 3: Launch and board (Launch step)

Final visualization of the brand DNA.

- **Hero visualizer**: Dark panel with lighting (blur) effects to highlight the brand
- **Meta information**: Shows match score and SEO-related tags
- **Actions**: Option to download the Brand Kit and restart the process

## 5. API implementation

### AI service (`src/services/aiService.js`)

The service supports three operating modes:

1. **Mock mode** (default): Uses predefined data, no API keys required
2. **OpenAI**: Integration with GPT-4 Turbo
3. **Google Gemini**: Integration with Gemini Pro

### Configuration

Environment variables are configured in `.env`:

```env
VITE_AI_PROVIDER=mock|openai|gemini
VITE_AI_API_KEY=your_api_key_here
```

### Response structure

The service expects the AI to respond with JSON in this format:

```json
{
  "identities": [
    {
      "name": "Brand name",
      "slogan": "Slogan here",
      "concept": "Visual concept here",
      "tags": ["Tag1", "Tag2", "Tag3"]
    }
  ]
}
```

### Error handling

- If the API fails, the system automatically uses mock data as a fallback
- Errors are logged to the console for debugging
- The user experience is not interrupted

## 6. Main components

### BrandAIModule (`src/components/BrandAIModule.jsx`)

Main component that manages the entire application flow.

**Main state:**
- `step`: Controls the current phase (1, 2, or 3)
- `projectDesc`: Project description
- `keywords`: Array of keywords
- `selectedPlan`: Selected plan (free, basic, premium, pro)
- `selectedIdentity`: Identity selected by the user
- `identities`: Array of generated identities
- `loading`: Loading state

**Main methods:**
- `handleGenerate()`: Starts identity generation
+- `handlePlanChange()`: Changes the plan and regenerates identities
- `handleKeywordAdd()`: Adds keywords to the array
- `removeKeyword()`: Removes a specific keyword
